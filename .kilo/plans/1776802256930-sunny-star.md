# Plan de Solución para Problemas en la App Colombia Open Data

## Fecha: 2026-04-21
## Estado: En Revisión

## Resumen de Problemas Identificados
La aplicación Flutter presenta varios problemas críticos que afectan la experiencia de usuario:
1. Desbordamiento visual (overflow) en cards de Departamentos y Atractivos Turísticos
2. Falta de navegación para regresar al dashboard
3. Imágenes faltantes en perfiles de presidentes
4. Error 404 en la sección de Festivos

## Análisis Detallado

### 1. BOTTOM OVERFLOWED BY 23 PIXELS en Cards
**Causa probable:**
- Las cards utilizan un layout fijo con `Column` que no maneja contenido variable
- El texto de descripción puede ser demasiado largo para la altura fija
- Falta de manejo responsive en el diseño de las cards

**Impacto:**
- Mal aspecto visual
- Contenido cortado o no visible

### 2. Falta de Navegación de Regreso
**Causa probable:**
- Las vistas de detalle no incluyen AppBar con botón de navegación
- go_router maneja back navigation, pero no hay UI para activarlo

**Impacto:**
- Usuario queda "atrapado" en vistas de detalle
- Mala UX sin forma de volver al menú principal

### 3. Imágenes Faltantes en Presidentes
**Causa probable:**
- La API de Colombia no tiene imágenes para todos los presidentes
- El código asume que siempre hay imagen disponible
- Falta manejo de casos null/empty en el campo image

**Impacto:**
- Avatares vacíos o errores de carga
- Aspecto inconsistente en la lista

### 4. Error 404 en Festivos
**Causa probable:**
- Endpoint `/Holiday/2024` puede no existir en la API
- El año por defecto (2024) podría no tener datos
- Falta validación del endpoint antes de consumir

**Impacto:**
- Sección completamente inutilizable
- Manejo de error genérico no informativo

## Soluciones Propuestas

### Arquitectura y Mejores Prácticas Aplicadas
- **Principio de Responsabilidad Única:** Separar lógica de UI y manejo de datos
- **Manejo de Errores Robusto:** Diferenciar entre errores de red, datos faltantes y validación
- **UI Responsive:** Usar layouts adaptativos en lugar de tamaños fijos
- **Defensive Programming:** Validar datos antes de renderizar
- **UX Mejorada:** Navegación intuitiva y estados de carga apropiados

### 1. Solución para Overflow en Cards
**Enfoque:**
- Reemplazar Column fijo por layout adaptable
- Usar Expanded y Flexible para distribución de espacio
- Implementar truncamiento inteligente de texto
- Agregar scroll interno si es necesario

**Archivos a modificar:**
- `lib/widgets/dashboard_card.dart`: Ajustar layout interno
- `lib/views/dashboard/dashboard_view.dart`: Verificar uso de GridView

**Implementación:**
```dart
// Cambiar de Column a estructura adaptable
Expanded(
  child: Column(
    mainAxisSize: MainAxisSize.min,
    children: [
      // Contenido adaptable
    ],
  ),
)
```

### 2. Solución para Navegación de Regreso
**Enfoque:**
- Agregar AppBar consistente con botón de back
- Implementar navegación programática con go_router
- Mantener consistencia en todas las vistas de detalle

**Archivos a modificar:**
- Todas las vistas de detalle (`*_detail_view.dart`)
- Implementar patrón común de navegación

**Implementación:**
```dart
AppBar(
  leading: IconButton(
    icon: const Icon(Icons.arrow_back),
    onPressed: () => context.go('/'),
  ),
  title: const Text('Detalle'),
)
```

### 3. Solución para Imágenes Faltantes
**Enfoque:**
- Implementar placeholders apropiados
- Usar CircleAvatar con iniciales cuando no hay imagen
- Manejar errores de carga de red gracefully

**Archivos a modificar:**
- `lib/views/president/president_list_view.dart`
- `lib/views/president/president_detail_view.dart`
- Crear widget reutilizable para avatares

**Implementación:**
```dart
CircleAvatar(
  backgroundColor: Colors.blue,
  child: Text(president.name[0].toUpperCase()),
)
```

### 4. Solución para Error 404 en Festivos
**Enfoque:**
- Verificar endpoint disponible en la API
- Implementar fallback con años alternativos
- Mejorar manejo de errores con mensajes específicos
- Agregar validación de respuesta antes de parsear

**Archivos a modificar:**
- `lib/services/holiday_service.dart`: Lógica de fallback
- `lib/views/holiday/holiday_list_view.dart`: Mejor UX para errores

**Implementación:**
```dart
// Intentar años alternativos si 2024 falla
final yearsToTry = [2024, 2023, 2025];
for (final year in yearsToTry) {
  try {
    return await ApiService.get('/Holiday/$year');
  } catch (e) {
    continue;
  }
}
```

- **Consistencia:** Mantener diseño coherente en toda la app
